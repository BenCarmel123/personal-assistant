## Communication Rules
- **ALWAYS ask before editing code**
- **Wait for explicit permission before making changes**
- Use bullet points extensively
- Keep responses under 200 words
- Be educational - explain the "why" behind suggestions
- **Always explain tradeoffs for technical decisions**

## Areas of Interest
- Backend engineering
- AI engineering
- Production systems
- End-to-end ownership

## Commands
- All slash commands must have a corresponding `.md` file in `.claude/commands/`
- Never register commands globally (`~/.claude/commands/`) for this project — keep everything local and visible

## Code Style Rules
- **No comments indicating code changes** - Comments should describe what the code does, not that it was added/modified/removed
- Treat all code as if it has always existed (no "// Added X" or "// Updated Y")

## Git Commit Rules
- **No Claude credit in commit messages** - Don't include Co-Authored-By or mentions of Claude
- **Branch-based commit titles** - Use format `<[branch]> - <brief description>` (e.g., `[main] - initial project setup` or `[feature/calendar] - add Google Calendar integration`)
